# Design System — exo

> Generated by `/design-consultation` on 2026-05-20, informed by a 3-vendor research council (Codex / Sonnet / Qwen3) and Michael's pick: both Topology (daily-driver) and Pulse (organic glance) as first-class views in one app.

## Product Context

- **What this is:** An AI-assisted infrastructure copilot. CLI today; web dashboard next. Diagnoses fleet state, proposes framework-grounded plans (K8s scheduler + DRS + 7 Rs), executes with consent gates, remembers outcomes.
- **Who it's for:** Homelab + selfhost peers running ~15–100 services across heterogeneous infrastructure (Proxmox + Docker + bare metal + edge devices) who can feel their sprawl is wrong but lack the framework to fix it.
- **Space:** homelab tooling / personal infrastructure / DevOps for the rest of us. Adjacent products: Beszel, Netdata, Komodo, Portainer, Glances, Grafana, Uptime Kuma, Proxmox UI, Tailscale Admin, Cockpit.
- **Project type:** primary surface is a self-hosted web dashboard; secondary surfaces are the CLI (Rich-rendered terminal UI) and the public leaderboard (static site at mgbuilds9.github.io/exo). DESIGN.md governs all three; CLI and leaderboard inherit the dashboard's tokens.
- **Memorable thing:** "I finally see my homelab as a system, not a list — and I see it both ways, scan-precise and feel-fast."

## Aesthetic Direction

This is a **two-view system**. Same data, same component vocabulary, two visual treatments accessible via a `Topology` ⇄ `Pulse` toggle in the topbar.

### Topology view — default, daily-driver
- **Direction:** operational blueprint, mid-century ops manual meets architectural drafting
- **Decoration level:** intentional — paper-texture background, deliberate ink lines, deliberate borders, but no ornament
- **Mood:** calm authority; cartographic precision; an opsroom in a 1960s aerospace facility; ink-on-paper trustworthiness
- **When you use this view:** every day. You're reading values, planning moves, executing on a runbook.
- **Council source:** Codex (light ops-room) + Qwen3 (zoom-not-click density) hybrid

### Pulse view — secondary, glance-mode
- **Direction:** organic system metaphor; the fleet as a living thing that breathes; relationships as gravity
- **Decoration level:** expressive — atmospheric halos, serif pull-quotes, animated state, single saturated highlight
- **Mood:** felt rather than read; the suffering of an overcommitted host is visible before you parse a number
- **When you use this view:** in the morning, after a long meeting, before sleep — to feel "is my fleet OK right now?"
- **Council source:** Qwen3 (force-directed organic) + Sonnet's "all entities visible, zoom not navigate"

### Shared across both views
- **Reject:** purple/violet gradients, centered hero with subtitle, 3-col icon-circle feature grids, gradient CTAs as default, system-ui/Inter/Roboto as primary, "Built for X" hero copy
- **Reference sites:** [Codex council citations](https://gethomepage.dev/), [Grafana Node Graph](https://grafana.com/docs/grafana/latest/visualizations/panels-visualizations/visualizations/node-graph/), [Proxmox UI](https://www.proxmox.com/en/products/proxmox-virtual-environment/overview), [Tailscale Admin](https://tailscale.com/), [Beszel](https://github.com/henrygd/beszel)

## Typography

The type system is **one humanist sans + one variable mono + one editorial serif** — shared across both views, with role differentiated by weight, size, and italic.

- **Display / hero / labels:** [Atkinson Hyperlegible](https://fonts.google.com/specimen/Atkinson+Hyperlegible) — designed for low-vision readability, scans at any zoom, never looks generic. Weights: 400 (default), 700 (emphasis).
- **Body / UI:** Atkinson Hyperlegible (same family — keeps the type system simple). Weight 400, line-height 1.45.
- **Data / numerics / IPs / ports / metrics:** [Recursive Mono](https://fonts.google.com/specimen/Recursive) — variable mono with weight axis (300/500/700) used for hierarchy *within* data cells. Tabular-nums enabled (`font-variant-numeric: tabular-nums`).
- **Editorial annotations (Pulse view only):** [PT Serif](https://fonts.google.com/specimen/PT+Serif) italic — warm serif for pull-quotes and "feel" labels. Subordinate to data; never used for chrome or buttons.
- **Code (CLI / runbook output):** Recursive Mono (same family as data; keeps install simple).
- **Loading strategy:** Google Fonts with `preconnect`; subset to `latin` + `latin-ext`. Total cold-load < 90 KB.

### Type scale (8/5 modular, base 13px)

| Role | Topology | Pulse | Notes |
|---|---|---|---|
| Hero / page title | 18px / 1.2 / 700 | 24px italic serif | Pulse uses serif italic for emotional weight |
| Section header | 13px / 1.4 / 700 (uppercase, letter-spacing 0.12em) | 10px / 1.6 / 500 (uppercase, letter-spacing 0.18em) | Pulse goes smaller and quieter |
| Body | 13px / 1.45 / 400 | 11.5px / 1.7 / 400 | |
| Data value | 11.5px Recursive Mono 500 | 10px Recursive Mono 500 | Tabular-nums on both |
| Caption / muted | 11px / 1.4 / 400 | 10.5px / 1.6 / 400 | |

## Color

The color system is **token-first, view-aware**. Semantic tokens are stable across views; their realized values differ between Topology (light default) and Pulse (dark default).

### Semantic tokens

| Token | Topology value | Pulse value | Meaning |
|---|---|---|---|
| `--surface` | `#F4F1EA` (warm paper) | `#0c0a09` (warm near-black) | The page background |
| `--surface-2` | `#FBF9F2` | `#08070b` | Cards, chassis, elevated surfaces |
| `--ink` | `#172026` | `#fafaf9` | Primary text |
| `--muted` | `#6B7682` | `#5c544c` | Secondary text, captions |
| `--grid` | `#D8D1C4` | `#181c20` | Background grid, dividers |
| `--selected` | `#005F73` (deep teal) | `#4ade80` (phosphor green) | Active selection |
| `--proposed` | `#CA6702` (warm orange) | `#4ade80` (phosphor green) | Proposed action / move |
| `--fault` | `#AE2012` (deep red) | `#d97757` (terracotta) | Critical / failure |
| `--healthy` | `#94A55C` (muted olive) | `#4ade80` | Healthy state |

### Brand chips (host-type, stable across views)

These are imported from the underlying product brand identities so they read fast for users who already know them:

| Type | Hex |
|---|---|
| Proxmox host | `#E57000` |
| Docker host | `#2496ED` |
| Raspberry Pi | `#C51A4A` |
| Windows | `#00A4EF` |
| Bare metal | `#94A3B8` |
| Tailscale-only | `#3b3b3c` |

### Light + dark mode

- **Topology** defaults to light; a dark variant inverts `--surface`/`--surface-2`/`--ink` and shifts `--grid` to dark.
- **Pulse** defaults to dark; a light variant exists for daytime use (background `#FAF8F0`, halos at `0.12` opacity).
- A single `prefers-color-scheme` listener handles auto-flip; a topbar toggle handles manual override.

## Spacing

- **Base unit:** 4px. All paddings, margins, gaps step in multiples of 4.
- **Density:** "comfortable" in inspector / sidebars (16px gutters, 20-24px section gaps); "compact" in canvas (8px gaps inside chassis, 6px in slots).
- **Scale:** `2xs(2)`, `xs(4)`, `sm(8)`, `md(16)`, `lg(24)`, `xl(32)`, `2xl(48)`, `3xl(64)`.
- **Container widths:** sidebar 320px (Topology), inspector 400px (Topology), no fixed widths in Pulse (full canvas).

## Layout

### Topology view

- Topbar: 56px, full width, contains brand + crumbs + view-toggle + meta-stats
- Left rail: 320px collapsible inventory tree (hosts / workloads / unmanaged)
- Center canvas: chassis blueprint (3 hosts side-by-side in this fleet; layout adapts for larger fleets via horizontal scroll)
- Right inspector: 400px "why this matters" panel for selected entity
- Background: paper grid at 32px

### Pulse view

- Topbar: 56px (same as Topology) — view-toggle anchored here
- Center: full-viewport force-directed canvas with atmospheric halos
- Annotations: serif pull-quotes float in the negative space (positioned by canvas, not grid)
- No persistent rails — selection opens a small floating inspector beside the selected node
- Background: radial-gradient warmth + grid at 48px / 0.3 opacity

### Border radius

- Cards / chassis: `4px` (Topology), `2px` (Pulse — sharper)
- Buttons: `2px` (both views)
- Pills / chips: `1px` (both — deliberately blocky, no pill aesthetic)
- Avatars / circles: `9999px` (full) — but used sparingly; the design avoids gratuitous circles

## Motion

Motion is **restrained and meaningful** in Topology; **slow and atmospheric** in Pulse.

### Topology

| Event | Duration | Easing |
|---|---|---|
| Hover | 120ms | ease-out |
| Panel open/close | 180ms | ease-in-out |
| Live value tick | 0ms | (no layout shift; just text swap) |
| Rebalance preview morph | 260ms | ease-in-out (pathfinding-style trace) |
| View toggle | 220ms | ease-in-out (cross-fade, not slide) |

### Pulse

| Event | Duration | Easing |
|---|---|---|
| Host breathing | 3.2-4.8s loop | ease-in-out (size + opacity, ±5%) |
| Migration drift | 3.3-4.0s loop | ease-in-out (particle x-translate) |
| Dependency edge highlight | 90ms | ease-out |
| Selection halo expand | 320ms | ease-out |

No entrance choreography on page load. No scroll-driven animation. No parallax.

## CLI styling (inherited)

The Rich-rendered terminal UI inherits the type tokens (Recursive Mono throughout) and the semantic color tokens mapped to ANSI:
- `--selected` → cyan
- `--proposed` → orange (256-color); falls back to yellow on 8-color terminals
- `--fault` → red
- `--healthy` → green
- `--muted` → bright_black

Tables in the CLI mirror the inspector's row style: dashed dividers, monospace data, left-aligned labels.

## Leaderboard styling (inherited)

The public leaderboard at `mgbuilds9.github.io/exo/` already uses a dark theme close to Pulse. It will be updated to:
- Switch to a `Topology`-style light default (matches the new system identity)
- Adopt Atkinson Hyperlegible + Recursive Mono
- Use the same semantic token names so the leaderboard and dashboard read as one product

## Decisions Log

| Date | Decision | Rationale |
|---|---|---|
| 2026-05-20 | Dual-view system: Topology default, Pulse glance | Michael picked "both" — practical daily-driver (A) + emotional health-glance (C). Validated by the 3-vendor council unanimously agreeing canvas-primary, then diverging on theme. |
| 2026-05-20 | Light default for Topology | Codex council voice's argument: every competitor (Beszel, Netdata, Komodo, Portainer, Grafana) defaults dark. Light is the boldest contrarian + maps + labels scan better. |
| 2026-05-20 | Dark default for Pulse | Atmospheric halos and breathing animations land better against warm near-black; the "felt" view wants dark. |
| 2026-05-20 | Atkinson Hyperlegible as primary | Designed for low-vision readability; never reads "generic AI default"; pairs across both views without modification. Replaces the Inter/Roboto convergence trap. |
| 2026-05-20 | Recursive Mono for all data | Variable mono = hierarchy within data cells without switching families. Tabular-nums by default. |
| 2026-05-20 | PT Serif italic only in Pulse | Editorial weight is the right tool for the "feel" view. Out of place in Topology. |
| 2026-05-20 | Reject Inter / Roboto / Open Sans / Poppins / Space Grotesk as primary | Convergence traps; every AI design tool picks these. Atkinson + Recursive is the contrarian + functional choice. |
| 2026-05-20 | Host-type brand colors imported | Proxmox orange, Docker blue, Pi red, Windows blue — users already recognize these; reuse them rather than invent. |
| 2026-05-20 | No purple/violet gradients, no centered hero, no 3-col icon-circle grids | AI design slop checklist; explicit rejections at the system level. |
