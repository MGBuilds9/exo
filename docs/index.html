How this is scored

For each candidate repo we fetch live GitHub signals via the gh CLI: stars, forks, push recency, recent commit count, recent release count, contributor count, open issue count, age, license, archive status. Four sub-scores each contribute to a 0–10 composite:

  • Maintenance — push recency + 90-day commit volume + recent releases. Heavy penalty for archived/disabled.
  • Popularity — stars with a stars-per-day momentum bonus.
  • Governance — contributor count, healthy issue backlog, age.
  • License fit — matches your constraint (any / permissive / no-AGPL / no-copyleft).

Weights for the default profile: maintenance 30%, popularity 30%, governance 25%, license fit 15%. Other profiles available in the CLI: reliable, cutting-edge, commercial.

Use it locally

The leaderboard lives here because it's a side-effect — the real product is the CLI. Get a personalized ranking with your own license constraint and weight profile:

exo recommend <capability> --weight-profile reliable --license no-agpl

Or, even better: describe what you're trying to build and let exo solve propose what capability you need, then chain to exo recommend automatically.
